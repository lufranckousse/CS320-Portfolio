# CS 320 Portfolio Reflection

## About This Repository

This repository contains work completed for CS 320: Software Test, Automation, and Quality Assurance. The project focused on building and testing a Contact Service using Java and JUnit. Through this project, I gained experience writing unit tests, validating software requirements, and improving software quality through testing.

---

## How can I ensure that my code, program, or software is functional and secure?

I ensure my software is functional by writing unit tests that verify every requirement before considering the project complete. During this course, I tested both valid and invalid inputs to make sure the application behaved correctly under different situations. I also validate user input by checking for null values, duplicate IDs, and invalid data before it is accepted. This helps make the software more reliable and secure.

---

## How do I interpret user needs and incorporate them into a program?

I begin by carefully reading the software requirements and breaking them into smaller tasks. Each requirement becomes part of the design and is verified with JUnit tests. This approach helps ensure that the software solves the customer's problem instead of simply functioning correctly.

---

## How do I approach designing software?

I approach software design by separating responsibilities into different classes and keeping methods focused on one task. I also design my code so it can be tested easily. Writing tests throughout development helped me catch mistakes early and produce cleaner, more maintainable code.
